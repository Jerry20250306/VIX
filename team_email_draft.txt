主旨：VIX 計算專案來了！用 Antigravity 助手一鍵搞定

各位夥伴，大家好：

VIX 計算專案的程式碼跟工具都整理好囉！現在大家可以直接用 Antigravity 助理來幫忙，不管是下載還是執行都超方便，不用再記那些死板板的指令。以下是幫大家整理的操作小指南：


1. 抓取專案跟更新 (交給 Antigravity 就好)

不需要自己去研究什麼 Git 指令，直接在 Antigravity 對話框跟它說一聲就行：

  - 第一次拿專案：對 Antigravity 說：「幫我從 GitHub 下載這個專案：[請在此處貼上 GitHub 專案網址]」
  - 同步最新版本：如果我說程式有更新，就跟助理說：「幫我同步最新的程式碼」或「幫我跑 git pull」。
    (AI 會自動幫你搞定底層同步，確保你的計算邏輯是最新的)


2. 環境自動準備

如果你是第一次跑，只要跟 Antigravity 說：「幫我安裝 VIX 專案需要的 Python 套件」，它就會自動幫你裝好 pandas, numpy, flask 這些工具，不用自己動手。


3. 資料放哪裡？

程式會自動找路徑，你只需要把資料丟到正確的資料夾：

  - 在專案底下開一個「資料來源」資料夾。
      - 行情資料：丟到 資料來源/J002-xxxxxx_YYYYMMDD/temp 裡面。
      - 比對資料 (PROD)：丟到 資料來源/YYYYMMDD 裡面。
  - 不確定路徑？直接問 Antigravity：「幫我檢查 20251231 的資料路徑對不對」，它會幫你檢查有沒有缺檔案。


4. 算完後的資料去哪找？

跑完計算之後，所有的結果都會放在「output」資料夾裡，主要有這幾種檔案：

  - 詳細比對資料 (驗證YYYYMMDD_Near/NextPROD.csv)：這是最完整的數據，包含我們算出來的值跟交易所 (PROD) 的值。如果你想用 Excel 檢查每一個時間點的計算細節，看這份就對了。
  - 差異摘要 (validation_diff_YYYYMMDD.csv)：這個檔案只會列出「算錯」的部分。如果你只關心哪裡有落差，直接開這個檔案，不用在幾萬筆資料裡大海撈針。
  - 驗證報告 (驗證報告_YYYYMMDD_全天數據.md)：這是文字版的總結報告，會直接告訴你今天的「一致率」是多少，以及主要的錯誤類型摘要。


5. 用口語叫助理動工

這部分最推薦，直接用說的：

  - 跑計算：直接說：「幫我跑 20251231 的 VIX 計算」或「看一下今天的數據有沒有問題」。
  - 看圖表報表：說：「幫我打開 Viewer 檢視器」，它會自動架好網頁並幫你開瀏覽器，讓你直接看哪些數據有差異。
  - 遇到報錯：執行失敗別緊張，直接把畫面或訊息丟給 AI：「這怎麼了？」，它會馬上診斷哪裡出問題。


6. 傳統老招 (如果你習慣自己來)

  - 跑計算：輸入指令 python run_batch.py --start 20251231 --end 20251231
  - 雙擊執行：直接點兩下資料夾裡的 start_viewer.bat。
