# VIX 自由探勘與行情重現面板 — 設計規格書

> **版本**: v1.0
> **狀態**: 設計討論完成 (待動工)
> **建立日期**: 2026-02-23

---

## 1. 介面佈局 (Page Layout)

採用 **「切換頁籤 (Tabs)」** 結構，將功能拆分為原本的「差異清單」與新的「自由探勘」。

### 1.1 頂部搜尋欄 (Search Bar)

- **基礎標的選擇**：`Term (Near/Next)`、`Strike (履約價)`、`CP (Call/Put)`。
- **時間精準定位**：
  - `Time`: HH:MM:SS 輸入框。
  - `[前一格] (-15s)` 與 `[後一格] (+15s)` 按鈕，方便連續觀察。
- **自動化跳轉**：
  - `[跳至上一個差異]`：針對該商品，自動搜尋前一個 Ours vs PROD 數值不符的時間點。
  - `[跳至下一個差異]`：針對該商品，自動搜尋後一個數值不符的時間點。

### 1.2 左右分欄 (Split View)

- **左側：數據快照與還原 (Analysis & Trace)**
- **右側：行情河流 (Continuous Tick Stream)**

---

## 2. 右側面板：行情河流系統 (Tick Stream Flow)

此面板採用 **「連續河流式 (Scheme B)」**，不再以 15s 為物理斷開點。

### 2.1 行情呈現與高亮

- **連續滾動**：列表包含所有原始 Tick，並在 Snapshot (00, 15, 30, 45秒) 時刻插入橫向分界線。
- **雙標籤標註 (並列顯示)**：
  - `[LAST]` (藍色標籤)：該 15s 區間內最後一筆有效的 Bid/Ask。
  - `[MIN]` (綠色標籤)：該 15s 區間內 Spread (價差) 最小的有效報價。
  - *註：同一筆 Tick 可同時掛上兩個標籤。*
- **動態擴充**：河流頂部與底部設有 `[讀取更早]` / `[讀取更晚]` 按鈕，點擊後向後端加載前後各 15s 的數據。

### 2.2 異常剔除視覺化 (1+2 方案)

- **灰階化 (Gray-out)**：所有被演算法判定為「無效」的 Tick 背景變暗/文字變灰。
- **原因代碼 (Error Codes)**：行末直接顯示剔除原因代碼，例如：
  - `[E1]`：價差大於上限 (e.g., > 24)。
  - `[E2]`：買價或賣價為 0。
  - `[E3]`：判定為離群值 (Outlier)。

---

## 3. 左側面板：數據快照與還原 (Calculation Path Trace)

### 3.1 數值比對區

- Side-by-side 呈現選定時間點的 `Ours` vs `PROD` 完整欄位。
- 高亮顯示不一致的數值（如 EMA, Gamma）。

### 3.2 算式還原區 (Trace)

- **公式展示**：顯示當前時間點的 EMA 計算公式。
- **真實數字帶入**：展示實際運算的數字流程。
- **決策路徑**：明確標出 $Q_{hat}$ 來源（是採用了哪個 SysID 的 $Q_{Last}$ 還是 $Q_{Min}$）。

---

## 4. 資料來源限制

- **純報價系統**：目前資料源僅提供 Bid/Ask，**不包含成交資訊 (Match Price)**。行情河流中將僅呈現報價異動。

---

## 5. 互動流程設計

1. **進入探勘**：使用者在頂部選定 `20251201`、`Near`、`22000`、`Call`。
2. **定位時間**：輸入 `10:00:00`，右側河流立即更新為該時間點前後 30s 的行情。
3. **觀察決策**：在河流中看到鮮豔的 `[LAST]` 與 `[MIN]`，並對照灰階的 `[Ex]` 報價，確認篩選邏輯是否符合預期。
4. **切換快照**：點擊右側河流中 `10:00:15` 的分界橫線，左側「算式還原區」立即更新為該時刻的計算細節。
5. **跨時追蹤**：點擊 `[跳至下一個差異]`，畫面自動滾動到下一個報價出錯的時刻，幫助定位是「算式錯誤」還是「行情抓取錯誤」。
